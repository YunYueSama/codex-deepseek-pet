# codex-deepseek-pet 任务计划

## 目标

构建并发布一只精致、透明的 Windows 桌面宠物，并提供 Codex 桌面版 v2 自定义宠物的生成与导入教程。宠物需要保持角色形象一致、清晰显示，支持全方向鼠标注视和丰富动作；可直接导入包只在全套检查通过后发布。

## 阶段

- [已完成] 检查源素材、项目状态和 Codex v2 自定义宠物格式。
- [已完成] 生成并验证更清晰、角色一致的高清待机素材，统一其余动作画布。
- [已完成] 优化桌宠默认尺寸、初始文案、常用动作清晰度和九方向注视。
- [已完成] 中文化文档，补充 Codex v2 生成与导入教程，执行视觉检查、自动化测试和 Windows 打包验证。
- [未完成] 可直接导入的 Codex v2 包；“审阅”动作行连续网络失败，未通过全套检查，因此不发布半成品。
- [已完成] 提交并推送更新到 GitHub 仓库。
- [已完成] 第三轮桌宠素材升级：统一全部动作画风，重做与基准风格不一致的开心、害羞和审阅。
- [已完成] 重新执行 12 动作同屏画风检查、透明边缘、动作触发和打包后截图检查。

## 约束

- 不覆盖或回滚用户已有改动。
- 不在源码、日志和文档中记录凭据或隐私数据。
- 角色图像沿用用户提供的蓝发鲸鱼女仆形象，保持服装、配色、轮廓和表情风格一致。
- `assets/你这吃白饭的蓝色大肥鱼.png` 是项目最高优先级的核心参考；所有动作的角色身份、画风、配色、线稿、服装细节和气质均以它为准，现有动作图只能作为姿势参考。
- 桌面版鼠标注视必须使用全局屏幕坐标，并覆盖中心及八个方向。
- Codex 自定义宠物包必须使用 `spriteVersionNumber: 2`、`1536x2288` 精灵表和 16 个顺时针注视方向。
- 保留呼吸、眨眼、歪头、害羞、开心、兴奋、跳跃、散步、单击、双击、拖拽和随机对白等动作。

## 问题记录

| 问题 | 尝试次数 | 处理结果 |
| --- | ---: | --- |
| 初次 `npm install` 超过过短的命令等待时间 | 1 | 使用合理的依赖安装超时重新执行。 |
| 中断安装后缺少 Electron 运行时文件 | 1 | 重新安装依赖，并单独下载运行时。 |
| Electron 默认下载源长时间无进度 | 1 | 通过 npm 镜像获取相同的官方发布文件。 |
| 最终验证和打包继承了 1 秒超时 | 2 | 测试与截图验证成功，使用更长超时单独重新打包。 |
| Electron Builder 26.15 在 Node.js 24 下生成配置前停滞 | 3 | 固定到 26.6.0；项目没有原生运行依赖，因此继续关闭 `npmRebuild`。 |
| Windows 占用失败构建生成的 `app.asar` | 1 | 将最终输出切换到已忽略的全新 `artifacts/` 目录。 |
| Electron Builder 25.1.8 带来 9 个仅开发环境的高危 tar 警告 | 1 | 升级到 26.6.0；生产依赖审计始终为零问题。 |
| npm 镜像不支持安全审计接口 | 1 | 使用官方 npm 仓库完成最终审计。 |
| Codex 手册辅助脚本访问公开文档时返回 403 | 1 | 改为核对本机 Codex 26.707.8168.0 的宠物模块及内置 `hatch-pet` v2 契约。 |
| 内置图片生成接口出现间歇性网络错误 | 2 | 保留未完成任务，继续处理其他独立动作行；网络恢复后单独重试，不写入空文件。 |
| v2“审阅”动作与高清“开心”“害羞”动作重试仍遇到网络错误 | 3 | 参考图编辑端点本轮仍失败；改用纯文本生成端点后，桌宠高清开心、害羞和单张审阅均成功。Codex v2 六帧审阅行仍保持未完成状态。 |
| CLI/API 备用路径缺少 `OPENAI_API_KEY` | 1 | 用户确认没有密钥；独立桌宠运行不需要 API，最终改用仓库内同批高清动作源，不再阻塞交付。 |
| 离线超分命令的 `Join-Path` 紧凑写法缺少空格 | 1 | 模型未启动、文件未覆盖；修正命令后完成对比，但超分候选因人物比例不一致被淘汰。 |
