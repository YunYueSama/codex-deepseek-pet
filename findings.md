## 产品方向

- Windows 透明无边框桌面宠物。
- 通过系统级鼠标坐标轮询，在鼠标位于其他应用时仍能持续注视。
- 使用中心加八方向的稳定区域驱动眼睛与身体动作。
- 同时提供 Codex 桌面版可识别的 v2 自定义宠物包。

## 素材

- 一张 `850x840` 角色参考图和七张 `384x416` 透明动作图。
- 当前核心参考为 `assets/你这吃白饭的蓝色大肥鱼.png`；它的蓝紫渐变长发、水润蓝眼、适中深色线稿、柔和赛璐璐上色、宝石蝴蝶结、袖口与裙摆金色刺绣、带白色褶边的鱼鳍耳饰和围裙鲸鱼图案，构成全部动作不可偏离的视觉基准。
- 动作图覆盖正面待机、左右跑步、思考、害羞、开心和兴奋。
- 七张透明动作图画布一致，切换姿势时不会改变窗口布局。
- 当前宠物在大窗口中接近原始像素上限，缩小默认窗口并换用高清素材可减少模糊感。
- 2026-07-13 再次逐张检查后确认：`idle.png`、`curious.png`、`excited.png` 为高清重绘；`happy.png`、`shy.png`、`run-left.png`、`run-right.png` 虽已统一为 `768x832`，主体仍带明显低分辨率锯齿，需要重新生成而非继续放大。
- 本轮新增动作选为挥手、审阅、困倦、跳跃和惊讶，覆盖问候、工作反馈、长时间空闲、双击庆祝与点击反馈等不同场景。
- 本地 v2 原始动作条均为 `2172x724`：四帧挥手、五帧跳跃和六帧等待/待机的单帧源尺寸足以用于桌宠；八帧失败动作中的角色过小，不作为高清桌宠素材。
- 原名 `running.png` 的动作条实际呈现正面思考循环，不具备明确的审阅语义；不会将其错误标记为审阅动作。`waiting.png` 第三帧的双手摊开姿势适合作为惊讶反馈。
- 连通组件提取可以在原始生成分辨率保留跨格鲸尾；均分裁切会产生尾巴残片，已被视觉检查拒绝。
- 九张新旧动作在 Electron 实际窗口中均完整显示；开心、害羞、挥手和惊讶保留动态瞳孔，睡眠、跳跃、跑步和审阅保留原画眼神。
- 用户复核指出本轮纯文本生成的 `happy.png`、`shy.png`、`review.png` 使用更细线、更高细节的插画风，与以 `idle.png` 为基准的圆润粗轮廓桌宠风格存在可见差异；三张均退回重做，不能仅凭清晰度和完整性通过。
- 用户进一步明确核心插画而非现有 `idle.png` 才是最高级画风参考；已中止一次未包含核心图的生成。随后以核心图作为第一参考的内置编辑请求仍返回网络错误，未产生文件。
- 用户没有 `OPENAI_API_KEY`，但独立 Electron 桌宠本身不调用 OpenAI API，因此运行、测试、打包和安装均不需要密钥；只有显式选择图片 API 的备用生成路径才需要密钥。
- 本地 Real-ESRGAN 候选虽能提高旧图边缘锐度，但无法消除旧版人物比例和线条差异，最终未进入正式素材。
- 最终 `happy.png` 使用同批唯一清晰笑脸姿势，并以独立开心跳动动画与兴奋动作区分；`shy.png` 使用同批合掌帧；`review.png` 使用同批专注思考帧。12 个动作入口现统一为同一套桌宠化角色比例、服装细节、发色和鲸尾设计。
- `assets/你这吃白饭的蓝色大肥鱼.png` 继续作为角色身份、配色、服装、鲸鱼主题和整体气质的最高级视觉基准；动作图只是在桌宠尺寸下的统一适配。

## Codex 自定义宠物格式

- 本机 Codex 26.707.8168.0 从 `%USERPROFILE%\.codex\pets\<宠物目录>` 读取本地清单。
- 每个宠物目录必须同时包含 `pet.json` 和 `spritesheet.webp`。
- v2 清单必须声明 `spriteVersionNumber: 2`；精灵表尺寸为 `1536x2288`，单格尺寸为 `192x208`。
- 精灵表包含 9 行标准动作以及 2 行、共 16 个顺时针注视方向；`000` 表示向上，不是正面静止。
- Codex 设置页提供“刷新”和“打开文件夹”，复制完成后可直接刷新并选择宠物。

## 工具链

- Electron 43.1.0 和 Electron Builder 26.6.0 已固定，以保证可复现构建。
- 当前网络环境构建 Electron、winCodeSign 和 NSIS 时使用镜像更稳定。
- 安装包没有受信任的商业代码签名证书，因此 Windows 可能显示 SmartScreen 提示。
